Solution Overview

The AI Gateway Suite (AIGS) provides a locally runnable AI stack composed of a React web application, an ASP.NET Core backend, and optional reverse proxies via Caddy or Nginx. Two deployment shapes are available: a layered architecture (Gateway.Api, Gateway.Application, Gateway.Domain, Gateway.Infrastructure, WebApp) and a self-contained SimpleGateway variant with built-in migrations, SQLite, and admin flows.

Repository Layout

src/
├─ Gateway.Api/             # Minimal APIs, DI wiring, health/metrics
├─ Gateway.Application/     # Use-cases, DTOs, validators, orchestration
├─ Gateway.Domain/          # Entities, value objects, interfaces (ports)
├─ Gateway.Infrastructure/  # EF Core, repositories, external LLM adapters
├─ SimpleGateway/           # Monolith alternative with SQLite + admin
└─ WebApp/                  # React frontend (Vite, Tailwind)

tests/
├─ Gateway.UnitTests/
└─ Gateway.IntegrationTests/

deploy/
├─ caddy/
├─ nginx/
└─ systemd/

Top-level scripts (setup-local.sh, start.sh, start-simple.sh, and their stop/status counterparts) assist with bootstrapping and lifecycle management.

Quick Start

SimpleGateway (all-in-one)

./setup-local.sh
./start-simple.sh
# visit http://localhost via Caddy

Layered stack (Gateway.Api + WebApp)

dotnet run --project src/Gateway.Api
cd src/WebApp && npm install && npm run dev
# API at http://localhost:5000, frontend at http://localhost:5173
