Project Overview

AIGS (AI Gateway Suite) is a comprehensive, locally runnable AI platform that combines a modern React frontend with a robust ASP.NET Core backend, seamlessly integrating with LM Studio for local AI inference.

What is AIGS?

In an era where artificial intelligence is transforming how we work and interact with technology, most AI solutions require sending your data to third-party cloud services. AIGS takes a different approach: it's a complete AI gateway solution designed for local-first deployments, meaning your data never leaves your infrastructure. Whether you're a privacy-conscious individual, a company with strict data governance requirements, or simply someone who wants to experiment with AI without recurring cloud costs, AIGS provides everything you need to run a production-grade AI chat system entirely on your own hardware.

Think of AIGS as your personal ChatGPT, but one that respects your privacy, runs on your machine, and gives you complete control over your conversations and data. It provides a full-featured chat interface with all the modern conveniences you'd expect: real-time streaming responses, organized conversation management, collaborative features like conversation sharing, and enterprise-grade security including user authentication, role-based access control, and optional end-to-end encryption.

At its core, AIGS integrates seamlessly with LM Studio's OpenAI-compatible API. This means you can run powerful open-source language models (like Llama, Mistral, or any model supported by LM Studio) directly on your laptop or server, and interact with them through a polished web interface that feels just as professional as any commercial offering.

The project offers two deployment modes to fit different use cases, from quick personal setups to enterprise-scale deployments:

  • SimpleGateway - A monolithic, all-in-one application that combines the entire stack into a single executable. Perfect for quick deployments, local development, and scenarios where simplicity and ease of use trump architectural complexity. You can have AIGS up and running in under 5 minutes with just a few shell commands.
  • Layered Architecture - A clean architecture implementation with separate Api, Application, Domain, and Infrastructure layers. This approach follows enterprise best practices and is ideal when you need testability, scalability, team collaboration, or plan to customize and extend the system significantly. Each layer is independent and can be tested, modified, or even swapped out without affecting the others.

What truly sets AIGS apart is its completeness. This isn't just a proof-of-concept or a minimal example - it's a full-featured platform with user management, conversation organization, chat templates, audit logging, webhook integrations, automated backups, and comprehensive monitoring. You get all the polish and features of a commercial SaaS product, but running entirely under your control.

Quick Start

Getting started with AIGS is designed to be as frictionless as possible. We understand that setting up a complex AI environment can often be a headache of dependencies and configurations. That's why we've streamlined the process into two distinct paths. Whether you just want to try it out in a few minutes or you're setting up a robust development environment, we have a path for you.

Before you begin, you'll need a few standard tools installed. These are the foundational building blocks that AIGS relies on to run efficiently on your local machine:

Prerequisites

  • .NET 8 SDK - The runtime for our high-performance backend. Download here
  • Node.js 18+ - Required to build and run the modern React frontend. Download here
  • LM Studio - The engine behind the AI. This needs to be running locally on http://127.0.0.1:1234 to provide the intelligence for the chat.
  • Caddy (optional) - A powerful, automatic HTTPS server that acts as a reverse proxy, simplifying how you access the application.

Option A: SimpleGateway (Recommended for Quick Start)

If you want to see AIGS in action immediately, the SimpleGateway mode is your best friend. It abstracts away the complexity of running separate frontend and backend processes. By using our convenience scripts, you can pull the code, configure the environment, and launch the entire suite with minimal effort. This is the ideal starting point for most users. 

# 1. Clone the repository
git clone <your-repo-url>
cd AI_Interface

# 2. Run setup script (installs/configures Caddy)
./setup-local.sh

# 3. Start the application
./start-simple.sh

# 4. Open your browser
# Frontend: http://localhost
# API: http://localhost/api
Default Login: Use username admin and password admin for your first login. Change this immediately in production!

To stop the application:

./stop-simple.sh

Check status:

./status.sh

Option B: Layered Stack (Development Mode)

For development with hot reload:

# 1. Start the backend API
cd src/Gateway.Api
dotnet run --urls="http://localhost:5000"

# 2. In a new terminal, start the frontend
cd src/WebApp
npm install
npm run dev

# Frontend will be available at: http://localhost:5173
# API will be at: http://localhost:5000
Note: In development mode, you'll need to ensure CORS is configured correctly in appsettings.json to allow requests from localhost:5173.

Deployment Modes

One size rarely fits all in software deployment. That's why AIGS is designed with flexibility in mind. Depending on your needs‚Äîwhether you're hacking on a new feature over the weekend or deploying a stable tool for your team‚Äîyou can choose the deployment model that makes the most sense for you. Both modes share the same core DNA but package it differently.

SimpleGateway (Monolithic)

The SimpleGateway mode is all about reducing friction. It bundles the frontend, backend, and database management into a single, cohesive unit. This approach minimizes the "moving parts" you need to manage, making it incredibly easy to deploy on a single machine or distribute to users who might not be technical experts.

Best for: Quick deployments, local development, demos, single-user setups

Characteristics:

  • Single executable with all layers combined
  • Built-in EF Core migrations
  • SQLite database by default
  • Admin tools and backup services included
  • Easy to deploy with included scripts
  • Perfect for air-gapped or offline environments

Use when: You need a quick setup, are running locally, or prefer simplicity over separation of concerns.

Layered Architecture

For scenarios demanding more rigor, the Layered Architecture mode shines. It strictly enforces separation of concerns, ensuring that your business logic, API endpoints, and infrastructure code remain distinct. This is the preferred mode for active development and enterprise environments, as it facilitates easier testing, cleaner code organization, and independent scaling of components.

Best for: Team deployments, production environments, scalability, testability

Characteristics:

  • Clear separation: Api ‚Üí Application ‚Üí Domain ‚Üí Infrastructure
  • Dependency Injection throughout
  • Easy to test individual layers
  • Supports horizontal scaling
  • Clean architecture principles
  • Can swap database/integrations easily

Use when: You need enterprise-grade architecture, plan to scale, have a team, or require extensive testing.

Tip: Start with SimpleGateway for proof-of-concept, then migrate to the layered architecture when you need more structure or scalability.
